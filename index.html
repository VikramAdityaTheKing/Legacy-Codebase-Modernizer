System Architecture

Legacy-to-modern AI pipeline

This tab focuses on the architecture only, so the diagram and design rationale are readable instead of buried in a long scroll.

System diagram

Design rationale

  • The Archeologist isolates business logic before translation so the Architect works from a normalized representation rather than raw source noise.
  • The Security Warden acts as a gating agent, not a passive scanner, which makes the architecture clearly multi-agent instead of sequential prompting.
  • The Validation Critic focuses on logic parity and drift, giving you a defensible answer to “how do you know the modernization is correct?”
  • The shared evidence layer captures artifacts needed for the written report, live demo, and any later backend implementation.
  • The browser implementation mirrors a Langflow-style orchestration graph even though it runs client-side for demo convenience.

Evaluation

Agent controls and test posture

This page explains how the system is evaluated and what knobs you can discuss during the interview.

Evaluation framework

  • Source detection confidence and correct sample-family selection.
  • Business-rule preservation from legacy code into FastAPI services.
  • Security Warden findings and remediation completeness.
  • Validation Critic parity score and drift-risk review.
  • Run cost estimate from approximate prompt and output token volumes.
  • Normal-case vs. test-case pass ratio in the generated review matrix.

Runtime settings

These controls are demo-level and influence how prompts are framed, how many tests are requested, and how verbose the generated deliverables become.
